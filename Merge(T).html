What is Merge Sort?

Merge Sort is a divide-and-conquer algorithm that divides the array into two halves, recursively sorts them, and then merges the sorted halves to produce the final sorted array.

How does Merge Sort work?

  1. Step 1: Divide the array into two halves.
  2. Step 2: Recursively divide each half until each subarray contains a single element.
  3. Step 3: Merge the subarrays by comparing elements from each half and combining them in sorted order.
  4. Step 4: Continue merging until the entire array is sorted.

Time Complexity of Merge Sort

The time complexity of Merge Sort is as follows:

  • Best case: O(n log n) - Always divides the array and merges sorted subarrays.
  • Average case: O(n log n) - The divide and merge process is consistently logarithmic.
  • Worst case: O(n log n) - The process always involves dividing the array and merging sorted halves.

Advantages of Merge Sort

  • Stable sort (preserves the order of equal elements).
  • Efficient for large datasets.
  • Has a predictable time complexity of O(n log n) in all cases.

Disadvantages of Merge Sort

  • Requires extra space for the temporary subarrays.
  • Not as efficient for smaller datasets compared to algorithms like Quick Sort or Bubble Sort.

Applications of Merge Sort

  • Used for large datasets where a stable sort is required.
  • Helpful in external sorting algorithms (sorting data too large to fit in memory).
  • Applied in situations requiring a stable, efficient sorting solution.

Merge Sort Example

Consider the array: [5, 3, 8, 4, 2]

The process of sorting it with Merge Sort:

  • Divide: [5, 3, 8, 4, 2] becomes [5, 3, 8] and [4, 2]
  • Further divide: [5, 3, 8] becomes [5] and [3, 8], [4, 2] becomes [4] and [2]
  • Merge: [3, 5, 8] and [2, 4] get merged to [2, 3, 4, 5, 8] (Array is sorted)
